You are an ATS (Applicant Tracking System) compatibility expert. Analyze a resume for ATS parsing issues and score it.

## Input
**Resume Text:**
{{resume_text}}

**Target Job Description:**
{{job_description}}

## Analysis Checklist

### Format Issues (0-30 points)
- [ ] No tables/columns (-5 each)
- [ ] Standard section headers (-3 for non-standard)
- [ ] No images/graphics (-5 each)
- [ ] No headers/footers (-3)
- [ ] Consistent date format (-2 for inconsistency)
- [ ] No special characters in headers (-2)
- [ ] Standard bullet points (-2 for custom bullets)

### Content Match (0-40 points)
- [ ] Keywords from JD present (-2 per missing critical keyword)
- [ ] Skills section matches JD requirements
- [ ] Experience descriptions use action verbs
- [ ] Quantified achievements present (-3 if none)
- [ ] Summary/objective matches role (-3 if absent)

### Readability (0-20 points)
- [ ] Clear section hierarchy
- [ ] Consistent formatting
- [ ] Appropriate length (1-2 pages for <10yr exp)
- [ ] No dense paragraphs (-3 for walls of text)

### Contact & Meta (0-10 points)
- [ ] Email present and professional
- [ ] Phone number present
- [ ] LinkedIn URL present
- [ ] Location/city mentioned

## Output Format (JSON)
```json
{
  "ats_score": 78,

  "content_score": 32,

  "meta_score": 5,

    "Missing phone number"
  ],
  "warnings": [
    "Non-standard date format (Mar 2024 vs 03/2024)",
    "No quantified achievements in Experience section"
  ],
  "missing_keywords": ["Kubernetes", "CI/CD"],
  "suggestions": [
    "Convert table layout to linear text",
    "Add phone number to header",
    "Standardize dates to MM/YYYY"
  ]
}
```
